  <files>src/store/cart.ts, src/components/ui/ProductCard.tsx, src/components/product/ProductInfo.tsx</files>
  <action>
1. Start dev server with `pnpm dev`.

2. **Test add-to-cart from ProductCard (homepage)**:
   - Open browser devtools console on http://localhost:3000
   - Click "ADD TO CART" on a product card
   - Check the cart drawer opens with the item
   - Click "ADD TO CART" on the SAME product again -- quantity should increment to 2 (not create duplicate)
   - Click "ADD TO CART" on a DIFFERENT product -- should add as new item

3. **Test add-to-cart from ProductInfo (detail page)**:
   - Navigate to a product detail page
   - Set quantity to 3 using the +/- buttons
   - Click "ADD TO CART"
   - **Known bug to verify:** ProductInfo.tsx calls `addItem` in a for-loop (`for (let i = 0; i < quantity; i++)`). Each `addItem` call checks for existing item and increments by 1. So adding quantity 3 of an item already in cart with quantity 2 should result in quantity 5 (not 3). This is technically correct but inefficient. If the user expects "set quantity to 3" to mean "I want 3 total", this is a UX bug.
   - **Decision:** Leave the addItem loop behavior as-is for now (it adds TO existing quantity). This matches standard e-commerce "add X more" behavior.

4. **Test cart drawer quantity controls (CartDrawer.tsx)**:
   - In the cart drawer, click "+" on an item -- quantity increments
   - Click "-" on an item -- quantity decrements
   - Click "-" when quantity is 1 -- item should be removed (the updateQuantity function calls removeItem when quantity <= 0)
   - Click the trash icon -- item should be removed

5. **Test cart total calculation**:
   - Verify the total shown in the cart footer matches sum of (price * quantity) for all items
   - Note: `total()` and `itemCount()` are computed functions, not derived state. They are called on every render via `total()`. Verify this doesn't cause hydration issues.

6. **Fix any issues found.** Common issues to watch for:
   - Hydration mismatch: CartDrawer uses `mounted` state to avoid SSR/client mismatch. Verify this works.
   - If `variantId` is undefined when adding from ProductCard (no variants), the cart will have an undefined key. Check that `product.variants.edges[0]?.node.id` always exists.

7. **Test localStorage persistence**:
   - Add items to cart
   - Check `localStorage.getItem('gymsnacks-cart')` in devtools -- should contain cart JSON
   - Refresh the page -- cart items should still be there
   - Navigate to a product page and back -- cart should persist
   - Open cart drawer after refresh -- items should display correctly
