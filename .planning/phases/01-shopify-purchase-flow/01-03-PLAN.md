**CRITICAL SECURITY FIX:** The current checkout API route accepts a raw GraphQL mutation string from the client and passes it directly to Shopify. This means any client can send ANY GraphQL mutation to the Shopify Storefront API through this endpoint. Fix this:

1. **Refactor src/app/api/checkout/route.ts** to accept line items (not a raw mutation):
   - Accept JSON body: `{ lineItems: Array<{ variantId: string; quantity: number }> }`
   - Validate: lineItems is an array, each item has variantId (string starting with "gid://shopify/") and quantity (positive integer)
   - Construct the checkoutCreate mutation SERVER-SIDE
   - Send to Shopify and return the checkout URL
   - Return proper error responses (400 for bad input, 500 for Shopify errors)

   New route structure:
   ```typescript
   export async function POST(request: Request) {
     const { lineItems } = await request.json();
     
     // Validate lineItems
     if (!Array.isArray(lineItems) || lineItems.length === 0) {
       return NextResponse.json({ errors: [{ message: "No line items provided" }] }, { status: 400 });
     }
     
     for (const item of lineItems) {
       if (!item.variantId || typeof item.variantId !== "string" || !item.variantId.startsWith("gid://shopify/")) {
         return NextResponse.json({ errors: [{ message: "Invalid variant ID" }] }, { status: 400 });
       }
       if (!Number.isInteger(item.quantity) || item.quantity < 1) {
         return NextResponse.json({ errors: [{ message: "Invalid quantity" }] }, { status: 400 });
       }
     }
     
     // Build mutation server-side
     const mutation = `mutation { checkoutCreate(input: { lineItems: [${lineItems.map(...)}] }) { ... } }`;
     // ... rest of Shopify call
   }
   ```

2. **Update src/store/cart.ts checkout()** to send lineItems instead of a raw mutation:
   ```typescript
   checkout: async () => {
     const { items } = get();
     if (items.length === 0) return;
     
     const lineItems = items.map((item) => ({
       variantId: item.variantId,
       quantity: item.quantity,
     }));
     
     const response = await fetch("/api/checkout", {
       method: "POST",
       headers: { "Content-Type": "application/json" },
       body: JSON.stringify({ lineItems }),
     });
     
     const data = await response.json();
     if (data.checkoutUrl) {
       window.location.href = data.checkoutUrl;
     } else if (data.errors?.length) {
       alert("Checkout error: " + data.errors[0].message);
     }
   }
   ```

3. **Clean up src/lib/shopify.ts**: Remove the `createCheckout` function -- it's dead code. The cart store's `checkout()` handles client-side checkout via the API route. Having a duplicate checkout function in a server-side file marked "Client-side only" is confusing.

4. **Fix Buy Now race condition in ProductInfo.tsx**: The current `handleBuyNow` calls `handleAddToCart()` then `setTimeout(() => checkout(), 500)`. The 500ms timeout is fragile. Instead:
   - After adding to cart, call checkout directly (no setTimeout)
   - The addItem is synchronous (Zustand set is sync), so the cart state IS updated immediately
   - The checkout() function reads from get() which will have the latest state
   
   Updated handleBuyNow:
   ```typescript
   const handleBuyNow = async () => {
     handleAddToCart();
     await checkout();
   };
   ```
